จนถึงตอนนี้ทุกข้อความเป็นสตริง แต่ WebSocket ก็พก **ไบต์** ได้อย่างมีความสุขเช่นกัน — รูปภาพ, เสียง, protocol buffer, สถานะเกมที่ถูกอัดแน่นในรูปแบบ binary บทเรียนสุดท้ายนี้ครอบคลุม frame สองรสชาติ, วิธีที่เบราว์เซอร์ส่ง binary data กลับมาให้คุณ และวิธีจับตาดูว่ามีข้อมูลรอจะออกไปอยู่เท่าไร

## text กับ binary frame

ในระดับโปรโตคอล ทุกข้อความ WebSocket เป็นได้ทั้ง **text** frame หรือ **binary** frame:

- **text** frame พก text ที่เข้ารหัสแบบ UTF-8 คุณสร้างมันโดยส่งสตริงเข้า `send()` และรับมันกลับมาเป็นสตริง
- **binary** frame พกไบต์ดิบโดยไม่มีสมมติฐานเรื่องการเข้ารหัส คุณสร้างมันโดยส่ง binary data เข้า `send()`

ฝั่งรับจะถูกบอกว่า frame นั้นเป็นชนิดใด จึงไม่มีวันกำกวมระหว่าง "สตริงสี่ตัวอักษร 1234" กับ "สี่ไบต์ดิบ" คุณเลือกรสชาติเพียงแค่ดูจากสิ่งที่คุณส่งเข้า `send()`

## การส่ง binary

`send()` รับ binary input สามแบบเพิ่มเติมจากสตริง:

```ts
// An ArrayBuffer — a fixed-length block of raw bytes.
const buffer = new ArrayBuffer(8);
ws.send(buffer);

// A TypedArray (or any ArrayBufferView) — a typed view over bytes.
const bytes = new Uint8Array([1, 2, 3, 4]);
ws.send(bytes);

// A Blob — file-like binary data, e.g. straight from an <input type="file">.
const blob = new Blob(['raw content']);
ws.send(blob);
```

ทั้งสามแบบถูกส่งเป็น binary frame เดียว `ArrayBuffer` และ `TypedArray` เป็นตัวเลือกปกติเมื่อคุณกำลังประกอบข้อความ binary ในcode ส่วน `Blob` โดดเด่นเมื่อข้อมูลมีอยู่แล้วในรูปไฟล์หรือมาจาก media API

## การรับ binary: binaryType

นี่คือการตั้งค่าหนึ่งเดียวที่สำคัญสำหรับ binary เมื่อ binary frame มาถึง `event.data` ควรเป็นอะไร — `Blob` หรือ `ArrayBuffer`? คุณตัดสินใจด้วย `ws.binaryType`:

```ts
ws.binaryType = 'arraybuffer'; // event.data will be an ArrayBuffer
// or
ws.binaryType = 'blob';        // event.data will be a Blob (the default)
```

ค่าเริ่มต้นคือ `'blob'` ตั้งเป็น `'arraybuffer'` เมื่อคุณต้องการอ่านไบต์ในทันทีและแบบ synchronous — เช่นเพื่อห่อมันใน `DataView` หรือ `Uint8Array` แล้วดึงฟิลด์ออกมา การอ่าน `Blob` เป็นแบบ asynchronous ดังนั้นสำหรับโปรโตคอล binary ที่ต้องการ latency ต่ำ `'arraybuffer'` มักจะเหมาะกว่า

```ts
ws.binaryType = 'arraybuffer';
ws.onmessage = (event) => {
  if (typeof event.data === 'string') {
    console.log('text frame:', event.data);
  } else {
    const view = new Uint8Array(event.data); // event.data is an ArrayBuffer
    console.log('binary frame, first byte:', view[0]);
  }
};
```

สังเกตว่า handler `onmessage` เดียวกันต้องแยกแขนงตามชนิดของ `event.data`: เป็นสตริงสำหรับ text frame และเป็น binary type ที่คุณเลือกสำหรับ binary frame

## การเฝ้าดู bufferedAmount

`send()` คืนค่าทันที แต่ไบต์ไม่ได้อยู่บนสายเสมอไป — ถ้าคุณส่งเร็วกว่าที่เครือข่ายจะระบายออก พวกมันจะกองพะเนินอยู่ใน buffer ภายใน ฟิลด์อ่านอย่างเดียว `ws.bufferedAmount` บอกคุณว่ายังมีกี่ไบต์ที่อยู่ในคิว:

```ts
if (ws.bufferedAmount < 1_000_000) {
  ws.send(nextChunk); // only push more if the buffer is not backing up
}
```

นี่คือสัญญาณ backpressure ของคุณ เมื่อสตรีม binary data ขนาดใหญ่หรือถี่ ๆ ให้ตรวจ `bufferedAmount` ก่อนส่งเพิ่ม ถ้ามันไต่ขึ้นเรื่อย ๆ แสดงว่าคุณกำลังผลิตเร็วกว่าที่การเชื่อมต่อจะพาไปได้ และการเรียก `send()` แบบไม่ดูตาม้าตาเรือจะทำให้หน่วยความจำบวม

## echo ArrayBuffer แบบสด

เดโมใช้ echo socket ในหน้าเว็บที่มี WebSocket API จริง จึงรันได้ทุกที่โดยไม่ต้องมี server มันตั้ง `binaryType` เป็น `'arraybuffer'`, ส่ง `Uint8Array` เล็ก ๆ และอ่านไบต์ที่ echo กลับมาออกมาตรง ๆ ในแอปจริงมีเพียงบรรทัดแรก — `new WebSocket('wss://your-server')` — ที่เปลี่ยนไป

export const binaryEchoCode = `(function () {
  // In-page echo socket with the same API as a real WebSocket.
  // In a real app: const ws = new WebSocket('wss://your-server').
  class EchoSocket {
    constructor() {
      this.readyState = 0;
      this.binaryType = 'blob';
      this.bufferedAmount = 0;
      setTimeout(() => { this.readyState = 1; if (this.onopen) this.onopen({}); }, 50);
    }
    send(data) {
      // Echo back respecting binaryType, like a real binary echo server would.
      let out = data;
      if (data instanceof ArrayBuffer || ArrayBuffer.isView(data)) {
        const buf = data instanceof ArrayBuffer ? data : data.buffer;
        out = this.binaryType === 'arraybuffer' ? buf : new Blob([buf]);
      }
      setTimeout(() => { if (this.onmessage) this.onmessage({ data: out }); }, 50);
    }
    close(code) { this.readyState = 3; setTimeout(() => { if (this.onclose) this.onclose({ code: code || 1000 }); }, 10); }
  }

  const ws = new EchoSocket();
  ws.binaryType = 'arraybuffer'; // we want raw bytes back, not a Blob
  ws.onopen = () => {
    const payload = new Uint8Array([10, 20, 30, 40]);
    console.log('sending bytes:', Array.from(payload).join(', '));
    ws.send(payload);
    console.log('bufferedAmount right after send:', ws.bufferedAmount);
  };
  ws.onmessage = (event) => {
    const isBinary = event.data instanceof ArrayBuffer;
    console.log('frame type:', isBinary ? 'binary (ArrayBuffer)' : 'text');
    if (isBinary) {
      const view = new Uint8Array(event.data);
      console.log('received bytes:', Array.from(view).join(', '));
    }
    ws.close();
  };
  ws.onclose = (event) => console.log('closed, code', event.code);
})();`;

สี่ไบต์ออกไปเป็น binary frame และกลับมาเป็น `ArrayBuffer` — เพราะเราตั้ง `binaryType = 'arraybuffer'` — ซึ่งจากนั้นเราอ่านผ่าน `Uint8Array` ถ้าเราปล่อย `binaryType` ไว้ที่ค่าเริ่มต้น `'blob'` `event.data` จะกลายเป็น `Blob` ที่ต้องอ่านแบบ asynchronous แทน

| Binary Type | ใช้เมื่อ | Trade-off |
|---|---|---|
| ArrayBuffer | ต้องการอ่านไบต์แบบ synchronous | ต้อง copy data ใน memory |
| TypedArray (Uint8Array) | มี structured binary data | ต้อง align กับ ArrayBuffer |
| Blob | file/media data ที่อ่านแบบ async | อ่านแบบ async เท่านั้น |

## Common Mistakes

**ลืมตั้ง binaryType ก่อน onopen**
อาการ:
- binary message มาถึง แต่ `event.data` เป็น Blob แทน ArrayBuffer
- code พยายาม `new Uint8Array(event.data)` แต่ fail เพราะ Blob ไม่ใช่ ArrayBuffer
- ตั้ง `ws.binaryType = 'arraybuffer'` ทันทีหลัง constructor ก่อน `onopen`

**ไม่ Check bufferedAmount — ส่งเร็วเกินไป**
อาการ:
- loop ส่ง chunk ขนาดใหญ่ติดกัน — `bufferedAmount` พุ่งสูง
- memory ฝั่ง client บวม network timeout
- check `if (ws.bufferedAmount < threshold)` ก่อนส่ง chunk ถัดไปเสมอ

> 💡 Real World Example
>
> **Binance/Bybit WebSocket Streams:**
> - ส่ง market data เป็น binary — compact กว่า JSON 3-5x
> - `binaryType = 'arraybuffer'` + `DataView` เพื่อ parse order book update latency ต่ำ
>
> **Figma Multiplayer:**
> - ส่ง drawing operation เป็น binary frame
> - ตรวจ bufferedAmount ก่อนส่ง batch operation เพื่อป้องกัน lag
